# 照片处理 GUI 工具计划文档

## 1. 目标

本计划文档用于定义项目从空仓库到首版可用工具的实施顺序、阶段目标与交付物。

## 2. 总体节奏

建议按 5 个阶段推进：

1. 项目基础设施阶段
2. GUI 骨架阶段
3. 时间修改功能阶段
4. 原片保留功能阶段
5. 测试与收尾阶段

## 3. 阶段计划

### 阶段一：项目基础设施

目标：

- 建立 Python 项目基本结构
- 建立 `.venv` 运行约束
- 明确依赖管理方式
- 准备启动入口和开发入口

交付物：

- 项目目录结构
- `.venv` 使用说明
- 依赖清单
- 应用启动脚本
- README 初稿

完成标准：

- 新开发者可以在 `.venv` 中启动项目
- 项目不会默认依赖系统全局 Python

### 阶段二：GUI 骨架

目标：

- 搭建主窗口
- 搭建两个主功能页
- 搭建基础表单、文件选择、结果区和状态区

交付物：

- 主窗口框架
- 时间修改页
- 原片保留页
- 通用消息提示与确认对话框

完成标准：

- 两个页面能独立展示
- 基本交互流可跑通但可暂时不接真实处理逻辑

### 阶段三：时间修改功能

目标：

- 实现文件扫描
- 实现时间偏移计算
- 实现逐文件预览
- 实现批量执行

交付物：

- 时间修改任务模型
- 预览表格数据
- 时间写回执行逻辑
- 结果统计与错误展示

完成标准：

- 用户可完成“选择 -> 配置 -> 预览 -> 确认 -> 执行 -> 查看结果”的完整流程

### 阶段四：原片保留功能

目标：

- 实现成片目录扫描
- 实现原片目录扫描
- 实现 basename 匹配
- 实现两种删除策略

交付物：

- 原片保留任务模型
- 保留/待处理预览
- 回收站模式执行
- 永久删除模式执行
- 结果统计与错误展示

完成标准：

- 用户可完整执行原片筛选与清理流程
- 高风险操作有完整确认链路

### 阶段五：测试与收尾

目标：

- 补齐单元测试和关键集成测试
- 完善错误提示
- 完善文档
- 做一次整体流程验证

交付物：

- 测试用例
- README 完整版
- 常见问题说明
- 已知限制列表
- 后续待办列表

完成标准：

- 两个核心流程可稳定运行
- 主要异常路径有可见反馈
- 文档可支持后续继续开发
- 后续扩展项已记录并明确当前不纳入首版范围

## 4. 优先级

优先级顺序如下：

1. `.venv` 约束与项目骨架
2. 时间修改预览
3. 时间修改执行
4. 原片保留预览
5. 原片保留执行
6. 测试与文档完善

原因：

- 先统一环境，避免后续依赖漂移
- 先做预览，降低执行阶段返工风险
- 删除类功能晚于预览类功能上线，更安全

## 5. 风险控制计划

### 5.1 环境风险

- 统一通过 `.venv` 启动和测试
- 禁止默认依赖系统 Python

### 5.2 文件处理风险

- 先做预览，不直接执行
- 批量处理单文件失败隔离

### 5.3 删除风险

- 默认优先回收站
- 永久删除需额外确认

### 5.4 平台风险

- 尽早验证 macOS 和 Windows 的文件时间写入差异
- 把平台差异收敛到基础设施层

## 6. 近期执行顺序

如果现在开始开发，建议按以下顺序直接落地：

1. 初始化项目目录和 `.venv`
2. 建立依赖管理与启动方式
3. 搭建 PyQt 主窗口和两个页面
4. 实现时间修改的数据模型和预览流程
5. 实现时间修改执行流程
6. 实现原片保留的数据模型和预览流程
7. 实现原片保留执行流程
8. 补测试与文档

## 7. 里程碑定义

### M1：项目可启动

- `.venv` 可用
- GUI 可打开
- 主界面结构完成

### M2：时间修改可用

- 可预览
- 可执行
- 可展示结果

### M3：原片保留可用

- 可预览
- 可执行
- 可展示结果

### M4：首版可交付

- 两个功能完整
- 文档完整
- 测试覆盖核心路径

## 8. 结论

本计划以“先环境、后骨架、先预览、后执行、先低风险、后高风险”为实施原则，适合当前空项目快速启动，并能把删除类风险控制在可接受范围内。
